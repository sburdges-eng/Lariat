# Lariat Cockpit — v1.5 + v2 modules

The kitchen-facing dashboard for The Lariat.

**v1.5 (line ops)**

- **Live Line Checks** — every station's checklist on iPad with pass/fail/par/have, persists to a local SQLite database, sign-off per station per shift. Each item has a one-tap **86** button right on the row.
- **Recipe Hub** — every recipe from the Lariat Recipe Book (Excel + PDF, deduped), searchable by name, ingredient, or allergen, with batch-scaler.
- **86 Board** — cooks 86 items with reason and qty, KM resolves them when restocked. Active 86s show as a red banner on the Today page.
- **Inventory Updates** — append-only log for mid-shift adjustments (prepped, received, used, counted). Recent updates surface on the Today page.

**v2 (loaded via ingest into SQLite)**

- **Analytics** — Toast item sales and monthly Shamrock spend (from workbooks).
- **Costing / BOM** — vendor prices, recipe costs, BOM lines, ingredient maps (from Master Costing workbook).
- **Order guide** — purchasing list from operations workbook.
- **Menu engineering** — margin vs popularity quadrant view (needs overlapping sales + cost data).
- **BEO** — banquets/events and prep tasks in SQLite.

Built as a Next.js 14 app that runs entirely on a laptop on your kitchen wifi. iPads connect over LAN.

## First run

1. Install Node.js (LTS) from <https://nodejs.org> if you don't have it.
2. **Double-click `Lariat Cockpit.command`** in this folder.
3. First run installs dependencies (≈1 minute). After that it's instant.
4. The Terminal window will print two URLs:
   - `http://localhost:3000` — open in Safari/Chrome on this Mac
   - `http://192.168.x.x:3000` — open in Safari on the iPad (any device on the same wifi)

To stop the server, hit **Ctrl-C** in the Terminal window.

## Daily use

- **Cooks** open the iPad URL, pick their name in the sidebar, tap a station, run the line check, and hit "Sign off this station."
- **You** open the laptop URL to see live status across all 6 stations on the Today page — red/yellow/green at a glance, who signed off, last update time, what's flagged.
- **Recipes** are searchable from any device. Tap a recipe to see ingredients with a batch scaler.

## Updating data

**Default source files** in this repo (override with env vars — see `OPERATIONS.md`):

| File | Purpose |
|------|---------|
| `XL/Lariat_Unified_Workbook.xlsx` | Line checks (BOH / BOH OOP sheets), setups, Recipe Book, Labor → staff, Toast sales |
| `XL/Lariat Recipe Book.pdf` | Optional PDF recipes (`LARIAT_PDF` if not beside the workbook) |
| `XL/Lariat_Master_Costing_*.xlsx` | Vendor prices, recipe costs, BOM, ingredient maps |
| `XL/lariat_operations_workbook_*.xlsx` | Order guide |
| `XL/Lariat_Analytics_Workbook.xlsx` | Monthly Shamrock spend trend |

**What each command writes**

| Command | Output |
|---------|--------|
| `npm run ingest` | `data/cache/line_checks.json`, `setups.json`, `recipes.json`, `staff.json` (from **Labor - By Employee**), `stations.json` (copied from `scripts/stations-seed.json`) |
| `npm run ingest:costing` | SQLite: `vendor_prices`, `recipe_costs`, `bom_lines`, `ingredient_maps`, `order_guide_items` |
| `npm run ingest:analytics` | SQLite: `sales_lines`, `spend_monthly` |
| `npm run ingest:all` | All of the above in order |
| `npm run validate-cache` | Validates JSON shape under `data/cache/` |

Restart the app after ingest if it is already running.

To export today's line checks, sign-offs, 86s, and inventory updates to a real `.xlsx` workbook (plus CSV fallbacks) for archive or audit:

```bash
npm run export                  # today
npm run export 2026-04-07       # specific date
npm run export:v2               # snapshot v2 tables (costing, sales, BEO, …) to exports/
npm run export:v2 2026-04-07    # same with dated filename
```

Daily ops export lands in `exports/lariat_YYYY-MM-DD.xlsx` (4 sheets: Line Checks, Sign-offs, 86 Board, Inventory) along with the per-table CSVs. The **v2 snapshot** adds `lariat_v2_snapshot_YYYY-MM-DD.xlsx` (imported costing, sales, spend, BEO, order guide, etc.). The xlsx steps need `python3` with `openpyxl` (`pip install openpyxl`); if that fails the CSVs are still written.

## Architecture

1. **Excel** is the human layer. You edit workbooks in Excel.
2. **JSON cache** (`data/cache/*.json`) is generated by `npm run ingest` for line ops and recipes (do not edit `stations.json` by hand — change `scripts/stations-seed.json` and re-run ingest).
3. **SQLite** (`data/lariat.db`) is the live ops write layer **and** the imported v2 read model (costing, sales, BEO).

```
Excel  →  ingest scripts  →  JSON cache / SQLite  →  Next.js
                                              ↓
                                    iPad UI (v1 write paths)
```

See **[OPERATIONS.md](OPERATIONS.md)** for backups, ingest env vars, and multi-location notes. **[SMOKE_TESTS.md](SMOKE_TESTS.md)** is a quick QA checklist.

## Auth & multi-location

- **Auth:** cook picker on line ops; optional **KM PIN** for money/costing: set env **`LARIAT_PIN`**, then open `/login-pin` once per browser session (or use the sidebar **Sign out** link). Omit `LARIAT_PIN` to leave analytics/costing unlocked (LAN trust only).
- **Multi-location:** All ops and v2 tables use `location_id` (default `default`). Use **`?location=`** on pages (e.g. `/inventory?location=west`) and APIs; add locations in SQLite `locations` before using non-default IDs. **`LARIAT_EXPORT_LOCATION`** / **`LARIAT_LOCATION`** filter `npm run export` (default `default`).

## Troubleshooting

**"Node is not installed"** — install LTS from nodejs.org and double-click the launcher again.

**Dependencies fail to install** — run `npm install` from Terminal in this folder. If `better-sqlite3` fails to build, run `xcode-select --install` and try again, or `npm rebuild better-sqlite3` after changing Node OS/arch.

**iPad can't reach the laptop** — both devices need to be on the same wifi network. Disable any "private wifi address" setting on the iPad. The laptop's IP can change — the launcher prints the current one each run.

**Recipes / line checks are out of date** — run `npm run ingest`.

**Want to wipe today's line checks** — delete `data/lariat.db`. The app will recreate it empty on next start. (Re-run `npm run ingest:costing` and `npm run ingest:analytics` to refill v2 tables.)
