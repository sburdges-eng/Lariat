# Slice 2.7 — Kitchen assistant action confirmation surfaces

Date: 2026-06-08
Branch: fix/lari-undo-toast

Goal
- Finish roadmap slice 2.7 by adding a 30-second undo surface after Kitchen assistant write actions.
- Use audit correction rows, not audit mutation, so the history stays append-only.
- Keep copy short and kitchen-native.

GitNexus grounding
- `kitchenAssistantPostHandler` blast radius is LOW upstream: only `POST` in `app/api/kitchen-assistant/route.js` calls it.
- `KitchenAssistantClient` blast radius is LOW upstream in GitNexus.
- GitNexus FTS is degraded on this repo, so source inspection stays the fallback for detailed path tracing.

Scope to ship
1. Add undo metadata to successful Kitchen assistant write responses when the action produced one concrete audit insert row.
2. Add `POST /api/kitchen-assistant/undo`.
3. Keep the undo window at 30 seconds.
4. Record undo as `audit_events.action='correction'` with `replaces_id` pointing at the original Kitchen assistant audit row.
5. Reverse the visible write for supported single-row actions:
   - `eighty_six`: mark the row resolved.
   - `inventory_updates`, `order_guide_items`, `equipment_maintenance`, `gold_stars`, `line_check_entries`: remove the just-created source row inside the same transaction as the correction audit row.
6. Do not offer undo for batch-style assistant actions that do not create one concrete row (`scale_recipe`, `beo_add_prep`, `generate_prep`).

Implementation notes
- New helper module should centralize:
  - undo eligibility detection
  - undo payload formatting
  - 30-second expiry check
  - source-table reversal + correction-row write
- `app/api/kitchen-assistant/route.js` should capture the audit row id returned by `postAuditEvent()` for undoable actions and include it in the JSON response.
- `KitchenAssistantClient.jsx` should show a toast-like card with:
  - what landed
  - time left
  - `Undo` button
  - terse success/error follow-up copy
- The toast should auto-expire after 30 seconds and clear on a new question.

Acceptance checks
- A successful 86 action returns undo metadata; clicking Undo resolves the row and writes a correction audit row linked by `replaces_id`.
- A successful line check action returns undo metadata; clicking Undo removes the created row and writes a correction audit row.
- Undo after 30 seconds is rejected cleanly.
- Second undo on the same action is rejected cleanly.
- Client shows the undo card, counts down, and sends the undo request when tapped.
- Roadmap row 2.7 is marked closed with the shipped behavior and test references.
