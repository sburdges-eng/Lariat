Design language used here

  • No AI clichés. No purple gradients, no robot icons, no chat bubbles, no "AI is thinking" spinners. LaRi's pilot light is a steady amber dot that beats slower when idle and faster when computing — borrowed from physical kitchen equipment (warmer lights, walk-in thermometers).
  • Stations are the nav. The horizontal rail at top represents the actual kitchen. Status colors are HACCP-meaningful (moss = pass, amber = watch, rust = breach) so a glance reads the whole line at once.
  • LaRi answers inline. The new db_query action result appears as a real table with provenance (audit #88241) — not as a separate "AI response card." This mirrors how LaRi grounds against deterministic facts; the prose framing is small and the table is the point.
  • Hold to commit writes. The walk-in corrective-action button fills with rust color over 320ms — a release before the fill completes cancels. Borrowed from iPad keyboard-shortcut hint UIs; safer than a confirmation dialog because it never interrupts the cook.
  • Manager tier visible at all times. The pill in the top-right turns copper when a PIN is active. Cook tier shows muted grey. A cook glancing at the screen always knows whether manager actions are unlocked.
  • Type ramp. Inter for prose, JetBrains Mono for every number / id / time. Numbers always tabular-aligned so a column reads like a thermometer scale.

What this proves

  • The new db_query tool (shipping in feat/lari-expansion-and-audit) has a natural home in the UI: results render as inline tables with source-attribution chips, not as bot-style messages.
  • The audit findings (nav-registry gaps, missing food-safety palette entries) are fixable without a wholesale rewrite — the redesign is additive.
  • A production migration would happen as a parallel route (/v2/...) behind a feature flag — same Next.js shell, same APIs, new component tree using the CSS-variable tokens defined at the top of this file.
